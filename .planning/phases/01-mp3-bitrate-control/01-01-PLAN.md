<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → /synth | Untrusted JSON body contains mp3_bitrate integer |
| CLI invocation → main() | Operator-supplied --mp3-bitrate argument |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | parse_server_fields: mp3_bitrate integer read | mitigate | Validate against static allowlist {32..320}; set sentinel -1 and return HTTP 400 for any value not in the list. No silent clamp. |
| T-01-02 | Denial of Service | synth_worker: integer forwarded to mp3enc_init | mitigate | Sentinel check at all parse sites prevents invalid values from ever reaching the encoder. Only valid allowlist values or 0-fallback reach audio_encode_mp3. |
| T-01-03 | Tampering | main(): --mp3-bitrate atoi with no bounds | mitigate | Post-atoi validation against same allowlist; exit(1) with error message if invalid. Prevents corrupted server state at startup. |
| T-01-04 | Information Disclosure | HTTP 400 response body | accept | Error message lists valid values explicitly. Acceptable for a local inference tool with no auth; does not expose sensitive state. |
</threat_model>
